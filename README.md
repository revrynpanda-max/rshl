# Victus Corporate Gateway — Sovereign RSHL Lattice (v21.0.0)

A high-fidelity, industrial-grade multi-agent ecosystem connecting the **Victus Core** to the **RSHL Lattice**. This system manages 11 autonomous AI agents organized into specialized corporate departments.

---

## 🏛️ Corporate Architecture

Unlike traditional AI "roundtables," the Victus Core operates on a **Departmental Business Model** managed by the **Oracle Overseer**.

### 🏢 Departments
- **Hardware Maintenance & Upkeep**: Led by **KAI**. Monitors RSHL lattice stability, hardware health, and geometric space.
- **Research & Development (R&D)**: Led by **Researcher**. Handles internal system audits and external owner projects (e.g., Annunaki/Mythology research).
- **Strategic Operations**: Led by **Analyst**. Manages project blueprints, task consolidation, and owner support.
- **Technical Core**: Led by **Kai Coder**. Responsible for codebase forensics, engine optimization, and hardware integration.

---

## 🔗 MemPalace Integration
The ecosystem is anchored to the **MemPalace Bridge**. Identities are no longer hardcoded; they are resolved dynamically via the **RSHL Lattice (Port 3333)**.
- **Neural Identity Resolution**: Bots query the database for real-time user names, roles, and "Truth Weights."
- **Social Awareness**: Agents use Discord bios and history to address you and your team with 100% data-driven accuracy.

---

## 🚀 Key Features

- **Departmental Autonomy**: Bots perform **Proactive Industrial Labor** during their work shifts, scanning history for unfinished tasks and auditing the core.
- **Collaborative Blueprinting**: Oracle collaborates with the Owner in DMs to finalize a strategic blueprint before igniting a departmental mission.
- **Strategic Mission Hub**: All project missions are contained within dedicated threads in the **#oracle-chat** channel.
- **Sonic-Parallel Voice (Leo)**: High-speed vocal processing with parallel STT, Biometric DNA verification, and real-time lattice memory recall.
- **Zero-Hardcode Identity**: Pure database-led social interaction. No static name placeholders.

## 🛠️ Setup & Requirements

- **Node.js 18+**
- **KAI Engine**: Running on port `:3333` (RSHL Lattice)
- **OpenJarvis Backbone**: Running on port `:8080`
- **Environment Variables**:
  - `OWNER_ID`: Your Discord User ID (Master Recognition)
  - `OWNER_NAME`: Your real name for corporate greeting
  - `GROQ_API_KEY`, `CEREBRAS_API_KEY`, `OPENAI_API_KEY`: Neural backbones

### 🚀 Running the Core

```powershell
.\run-oracle-discord.ps1
```
